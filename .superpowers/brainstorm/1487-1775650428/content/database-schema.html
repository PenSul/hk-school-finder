Database Schema

expo-sqlite tables -- derived from the three government API datasets + user calendar events

SQLite Tables & Indexes
schools (K-12 from EDB API)
PK school_no TEXT NOT NULL
name_en TEXT
name_tc TEXT
category_en TEXT
category_tc TEXT
address_en TEXT
address_tc TEXT
IDX school_level TEXT
IDX district TEXT
IDX finance_type TEXT
IDX religion TEXT
IDX session TEXT
IDX students_gender TEXT
telephone TEXT
fax TEXT
website TEXT
latitude REAL
longitude REAL
hei_institutions (122 from ASFPS API)
PK objectid INTEGER NOT NULL
IDX facility_name_en TEXT
facility_name_tc TEXT
address_en / address_tc TEXT
telephone / fax / email TEXT
website TEXT
latitude / longitude REAL
ugc_programmes (~1031 from UGC API)
PK objectid INTEGER NOT NULL
IDX university_en TEXT
university_tc TEXT
IDX programme_name_en TEXT
programme_name_tc TEXT
IDX level_of_study_en TEXT
IDX mode_of_study_en TEXT
latitude / longitude REAL
calendar_events (user-created + seeded demo data)
PK id TEXT NOT NULL UUID
title TEXT NOT NULL user-entered name
description TEXT optional notes
IDX event_date TEXT NOT NULL ISO 8601 date
event_time TEXT optional HH:MM
IDX category TEXT NOT NULL poa | kg | open_day | sspa | custom
school_no TEXT optional FK to schools
reminder_enabled INTEGER DEFAULT 0 boolean flag
is_seeded INTEGER DEFAULT 0 1 = demo data, 0 = user-created
db_metadata (tracks seed state for update checks)
PK key TEXT NOT NULL 'schools_count', 'hei_count', 'programmes_count', 'last_seed_ts'
value TEXT NOT NULL record count or ISO timestamp
PK = Primary Key
IDX = Indexed column (used in WHERE/LIKE clauses)

Design Notes

  • All column names are snake_case -- mapped from the API's UPPER_CASE fields during seeding
  • calendar_events supports full CRUD -- user-created events + seeded demo data (distinguished by is_seeded flag)
  • db_metadata tracks record counts per table -- on subsequent launches, compare with API response count to detect updates
  • No foreign keys between school tables -- the 3 datasets are independent with no shared keys. UGC programmes link to universities by name, not by ID
  • Text columns for flexible data -- API values vary (e.g., "NOT APPLICABLE" vs empty for religion). Handle at the repository layer, not schema level
