Organization Principles

  • Feature-grouped components under components/ -- each feature has its own folder. Shared cross-feature components live in shared/
  • Data layer is separate -- db/, repositories/, stores/, and hooks/ are top-level because they're cross-cutting
  • Route files stay thin -- app/ route files import and compose components from src/components/. Screens are assembled, not authored, in route files
  • Types co-located by domain -- not by file type. types/school.ts covers School, HEI, and Programme together
  • Constants centralized -- colour tokens, badge maps, district/religion lists. No magic values in components
