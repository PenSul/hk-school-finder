Key Points

  • SQLite is the single source of truth for all school data -- fetched once, queried locally
  • Zustand manages transient UI state (filters, compare selections) with no persistence needed
  • MMKV persists only small user preferences (language, theme, shortlist IDs, status tracker stages)
  • React Context limited to theme and language providers (low-frequency updates)
  • Repository pattern encapsulates all SQL -- hooks and components never see raw queries
  • Calendar events stored in SQLite with full CRUD -- seeded from bundled demo JSON on first launch
